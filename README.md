# Wheel Service — MVP сайта

Первый рабочий вариант сайта для Wheel Service: тёмный индустриальный дизайн, графитово-жёлтая палитра, реальные фотографии сервиса и дисков, адаптивная вёрстка и SEO-структура.

## Что реализовано

- Главная страница с hero-блоком и основными CTA.
- Отдельные страницы: услуги, цены, работы, контакты и запись.
- 9 SEO-страниц услуг в `/services/[slug]`.
- Порошковая покраска выделена как главное направление.
- Реальный прайс R13–R23 и дополнительные работы.
- Интерактивный блок «До / После».
- Галерея из загруженных фотографий.
- Яндекс.Карта с lazy loading.
- Быстрые ссылки: звонок, WhatsApp, Telegram, маршрут.
- Мобильная sticky-панель действий.
- Форма без backend: формирует сообщение и открывает WhatsApp.
- Schema.org разметка локального автомобильного бизнеса.
- `sitemap.xml`, `robots.txt`, Open Graph и manifest.
- Опциональное подключение Яндекс Метрики через `.env.local`.
- Базовый шаблон политики конфиденциальности.
- TypeScript strict, ESLint, Prettier и Stylelint.

## Запуск локально

Нужен Node.js 20 или новее.

```bash
npm install
npm run dev
```

Откройте:

```text
http://localhost:3000
```

Проверка production-сборки:

```bash
npm run typecheck
npm run lint
npm run stylelint
npm run build
npm run start
```

## Настройка перед публикацией

Создайте `.env.local`:

```env
NEXT_PUBLIC_SITE_URL=https://ваш-домен.ru
NEXT_PUBLIC_YANDEX_METRIKA_ID=номер_счётчика
```

`NEXT_PUBLIC_YANDEX_METRIKA_ID` можно оставить пустым — тогда аналитика не загружается.

## Где менять данные

- Контакты, адрес, часы, карта: `src/shared/config/site.ts`
- Категории и SEO-страницы услуг: `src/entities/service/data.ts`
- Цены: `src/entities/service/prices.ts`
- Галерея: `src/entities/service/works.ts`
- FAQ: `src/shared/config/faq.ts`
- Фирменные цвета: `src/app/globals.scss`
- Изображения: `public/images/`

## Как работает форма

Форма не требует backend и не хранит данные на сервере. После проверки полей она создаёт готовое сообщение и открывает WhatsApp первого номера из `siteConfig.phones`.
